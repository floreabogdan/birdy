{{template "icon-info"}}You probably need fewer of these than you think

An address configured on an interface already becomes a route, via the direct protocol, and iBGP carries it to your other routers. You do not need a static route for a subnet that lives on a router running birdy.

Static routes are for what is left over: a subnet behind something that does not speak BGP, or a route to a far router’s loopback so an iBGP session peering on loopbacks can resolve its next hop. To announce an aggregate, use a prefix set with originate instead.
