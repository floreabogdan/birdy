{{template "icon-info"}}How this works

Prefix holders publish ROAs saying which AS may originate their prefixes. BIRD does not fetch those itself: an RTR server feeds it a ROA table, and an import policy calls roa_check on every route.

Each route comes back valid, invalid or unknown. Only invalid means a ROA exists and contradicts the origin. Unknown is most of the internet and must never be dropped.

Start in log only: invalid routes get tagged RPKI_INVALID and still used, so you can count them first. Move to reject invalid once the number looks sane. Running a local validator (Routinator, StayRTR, rpki-client) beats depending on someone else’s.
