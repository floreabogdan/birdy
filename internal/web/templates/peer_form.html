BGP makes a peer prepend its own AS number before it announces a route, so every path this peer sends should start with AS{{if .Peer.RemoteASN}}{{.Peer.RemoteASN}}{{else}}…{{end}}. If it does not, they are misconfigured or leaking someone else's routes. Turn this off for an IXP route server: a route server forwards routes without prepending itself, so leaving this on would reject everything it sends.
The second switch checks the origin AS instead: it accepts a prefix only if this peer announced it as their own. Use it when you sell someone transit but do not carry their downstreams. Their own prepending still works. To carry their downstreams too, leave it off and point an import policy at an AS set holding their expanded AS-SET.
